What does it mean to 'display:inline' vs 'display:inline-block'?

Both are CSS terms that determine how an element behaves (layout wise). Originally I thought they could only be applied to images, but they can be used on any HTML element.

Display:Inline

This means the element will sit in line with other elements. It doesn't start on a new line and you can't set the width or height.

Often used for:

  • Text level elements (like span)
  • Styling part of a sentence without breaking layout
  • Keeping elements flowing naturally with other text

Display:Inline-Block

Similar to 'display:inline' in the sense it still sits in line with other elements, but with more flexibility. You can set the width, height, padding and margins.

Often used for:

  • Buttons that need custom size or spacing
  • Navigation links with padding or hover effects
  • Images where size or alignment need to be changed
  • Form inputs (like text fields or submit buttons)

In short, both are ways to control how elements behave on a webpage. 'display:inline' is great for small elements that flow naturally with text, but you won't have much control over size. 'display:inline-block' provides the side by side layout as well as the ability to set width, height and spacing.

Analogy

In one of my reflections I think I compared HTML to dog breeds. If HTML is the Border Collie (structure and smarts) and CSS is the Golden Retriever (style and friendliness) then JavaScript could either be the German Shepherd (alert, acts on command) or the owner maybe (calling the shots, telling the others what to do and when).

Control Flow and Loops

If we use a round of golf as an example,

The Control Flow would be the steps:

  1. Tee off from the first hole
  2. Hit the ball towards the green until you reach it
  3. Putt the ball until it goes in the hole
  4. Move to the next hole

You might also have conditions:

  • If the ball lands in the bunker, use a sand wedge
  • If you're on the green, use a putter

Loops happen when something repeats until a condition is met, like repeatedly hitting the ball until it gets in the hole.

The DOM

The DOM (Document Object Model) is how JavaScript sees and interacts with a webpage.

It turns HTML elements into objects that JavaScript can then access and change.

This week we did the JavaScript Cafe Challenge. It involved 'building' a web app then using the DOM to update elements based on customer interaction. We used JavaScript to randomly generate orders, display the orders and update stock counts accordingly (in real time, without having to refresh the page).

Arrays vs Objects

Arrays and objects are both ways of storing data in JavaScript, but are used differently.

Arrays are ordered lists, whereas objects are more like containers. During the JavaScript Cafe Challenge we used arrays to store lists of things - like customer orders, and objects to store information about each product - like stock and price.

Functions

Functions are reusable blocks of code that do a specific thing. You write the steps once then can run the function repeatedly without having to rewrite it each time.
